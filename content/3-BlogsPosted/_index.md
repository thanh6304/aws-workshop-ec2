## Blog 1 - AWS Lambda – Running Code Without Managing Servers

This blog introduces AWS Lambda as one of AWS's core Serverless computing services. It explains how Lambda executes code in response to events without requiring developers to provision or manage servers. The article covers Lambda's event-driven architecture, automatic scaling capabilities, pay-per-use pricing model, and seamless integration with services such as Amazon S3, Amazon API Gateway, Amazon DynamoDB, Amazon EventBridge, and Amazon SQS. It also discusses common use cases, the advantages of Serverless applications, and situations where Amazon EC2 or Amazon ECS may be more suitable solutions.

## Blog 2 - AWS Secrets Manager, Parameter Store, or AppConfig?

This blog explores three AWS services that help developers securely manage application secrets, configuration values, and feature settings: AWS Secrets Manager, AWS Systems Manager Parameter Store, and AWS AppConfig. It explains the purpose of each service, compares their features, pricing models, and common use cases, and discusses when each solution is the most appropriate. Through this comparison, readers can better understand how to choose the right service for managing credentials, configuration data, and application deployments in different AWS environments.

## Blog 3 - AWS Organizations Security

This blog focuses on the importance of protecting AWS Organizations from unauthorized account removal and maintaining security in a multi-account AWS environment. It introduces the risks associated with the `organizations:LeaveOrganization` API, explains how Service Control Policies (SCPs) help enforce security boundaries, and highlights AWS best practices such as applying the Principle of Least Privilege, securing Root Accounts with Multi-Factor Authentication (MFA), and implementing Break Glass procedures. The article also shares key lessons learned about governance, centralized security management, and cost control across AWS accounts.
