## Overview

The Dashboard is the primary interface of the AI Supply Chain Control Tower, providing operational metrics and AI-generated insights.

In this section, you will verify that the Dashboard correctly displays data from the Backend API and the AI processing pipeline.

---

## Architecture

```text
Amazon RDS
      │
      ▼
Backend API
      │
      ▼
API Gateway
      │
      ▼
Frontend (AWS Amplify)
      │
      ▼
Dashboard
```

---

## Objectives

After completing this section, you will:

- Verify the Dashboard.
- Validate KPI metrics.
- Verify AI-generated insights.
- Test data refresh functionality.

---

# Step 1. Open the Dashboard

Open the application deployed with AWS Amplify.

Sign in using a valid user account.

After authentication, the Dashboard should be displayed.

---

# Step 2. Verify KPI Metrics

Confirm that the Dashboard displays:

```text
Total Orders

Inventory Level

Active Shipments

Suppliers

Delayed Orders
```

Compare these values with Amazon RDS or Amazon Athena query results.

---

# Step 3. Verify Data Tables

Review tables such as:

```text
Inventory

Orders

Shipments

Suppliers
```

Verify that:

- Data loads successfully.
- No records are missing.
- Displayed values match the database.

---

# Step 4. Verify AI Insights

Review the **AI Insights** section.

Confirm that it includes results such as:

```text
Low Stock Prediction

Demand Forecast

Supplier Performance

Shipment Delay Analysis
```

Ensure these insights are generated by Amazon Bedrock based on the latest available data.

---

# Step 5. Verify Data Refresh

Create a new order through the Backend API.

After the AI pipeline completes:

- Refresh the Dashboard.
- Verify that KPI values are updated.
- Confirm that AI Insights reflect the new information.

---

# Step 6. Verify Error Handling

Simulate a Backend API failure.

Verify that the Dashboard:

- Displays a user-friendly error message.
- Does not become unresponsive.
- Allows the user to retry the request.

---

# Step 7. Verify Responsive Design

Open the Dashboard on:

- Desktop
- Tablet
- Mobile device

Verify that:

- The layout remains responsive.
- No UI elements overlap.
- Charts and tables display correctly.

---

# Step 8. Verify Performance

Review:

- Dashboard loading time.
- Backend API response time.
- AI Insights rendering time.

Ensure the overall user experience remains smooth.

---

## Best Practices

- Load only the data required for the initial view.
- Use pagination or lazy loading for large datasets.
- Display loading indicators while data is being fetched.
- Provide meaningful error messages.
- Refresh Dashboard data after AI processing completes.

---

## Verification

Verify that:

- KPI metrics display correctly.
- Data tables are accurate.
- AI insights are generated successfully.
- Dashboard updates after new data is processed.
- The interface works across different devices.

---

## Expected Outcome

After completing this section, you have:

- Verified the Dashboard functionality.
- Confirmed accurate KPI visualization.
- Validated AI-generated insights.
- Ensured the user interface is ready for production use.
